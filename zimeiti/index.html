分层时序自动更新机制

为了节约 token 并保证文章质量，系统按温哥华时间（PDT）分三层时序运行：

① 数据采集（PT 04:00 / 16:00）

每天温哥华时间凌晨 4 点和下午 4 点，CST 19:00 和 07:00，queue_engine.py scan 自动扫描新增的视频脚本：

  • 归类到现有话题 → 如果带新观点/数据点，入队为 update_topic 任务
  • 未匹配任何现有话题 → 入队为 new_topic 任务（高优先级）
  • 已处理过的脚本通过 SHA256 hash + HTML marker 双重去重，不会重复

注意：采集阶段不修改任何 HTML，只把任务写入队列。

② 编辑时段（PT 20:00 – 09:00，每 30 分钟一次）

CST 11:00–23:30 每 30 分钟 + CST 00:00 一次，edit_apply.py apply 从队列取一个任务：

  • new_topic 任务：创建新话题大全 HTML（自动编号 08、09、10...）
  • update_topic 任务：把新观点追加到现有文章的"新观点补充"section + 更新日志
  • 新话题优先处理

③ GitHub Pages 同步（PT 18:00）

CST 09:00 自动 git commit + push，GitHub Pages 1–2 分钟内自动部署。

Cron 表达式（系统时区 CST）

动作Cron 表达式对应 PT 时间
扫描 outputs0 19 * * *≈ PT 04:00
扫描 outputs0 7 * * *≈ PT 16:00（次日）
编辑时段*/30 11-23 * * *PT 20:00 – 14:30
编辑收尾0 0 * * *≈ PT 09:00
同步 Pages0 9 * * *≈ PT 18:00

脚本源代码

所有 Python 脚本都部署在 GitHub Pages 上，可直接下载使用：

  • queue_engine.py — 数据采集引擎（04/16 点跑）
  • edit_apply.py — 编辑写入引擎（每 30 分钟跑）
  • patch_v3_anchors.py — 给 v3 文章补 anchor（一次性）

本地仓库：~/zimeiti/

手动操作

手动扫描

python3 ~/zimeiti/scripts/queue_engine.py scan

查看队列

python3 ~/zimeiti/scripts/edit_apply.py list

手动处理一个任务

python3 ~/zimeiti/scripts/edit_apply.py apply

重置 hash 表（强制重新扫描所有脚本）

rm ~/zimeiti/scripts/.processed.json
python3 ~/zimeiti/scripts/queue_engine.py scan
