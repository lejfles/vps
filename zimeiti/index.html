v4 学术级规范

每篇文章遵循 LLM wiki 学术级规范：

  • Abstract 段：200-300 字 LLM wiki 风格摘要
  • Table of Contents：自动生成章节跳转
  • H2/H3/H4 章节：10-15 个 H2，30-50 个 H3，理论 + 数据 + 方法论 + 案例分层
  • 业界权威基线对比：每个话题对照学术/行业/平台官方 3 个最高权威源
  • 12-15 条参考文献：带 URL + 期刊/媒体名 + 权威标签
  • 风险与争议：明确标注哪些是共识、哪些是开放问题
  • 未来 12 个月观察：4+ 个可证伪的预测变量
  • 新观点补充段（H2）：所有历史 video 脚本触发的观点汇总

分层时序自动更新机制

① 数据采集（PT 04:00 / 16:00）

CST 19:00 和 07:00，queue_engine.py scan 自动扫描新增视频脚本，归类入队。

② 编辑时段（PT 20:00–09:00，每 30 分钟）

CST 11:00–23:30 + 00:00，edit_apply.py apply 每 30 分钟处理一个任务。新观点追加到对应文章的"新观点补充（H2）"段。

③ GitHub Pages 同步（PT 18:00）

CST 09:00，sync_to_pages.sh 自动 git commit + push。
