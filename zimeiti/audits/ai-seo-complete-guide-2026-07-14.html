1 核心理念：AI SEO ≠ 堆关键词 Core Philosophy

AI 时代最大的认知误区：把 SEO 简化为"批量写文章堆关键词"。 真正的 AI SEO 是可控的内容资产系统——以商业目标为中心、以证据为支撑、以结构化交付物为成果、以商业风险为红线。

1.1 AI SEO 的本质是「内容资产生产」

🎯

商业目标优先

明确是流量、信任、AI 引用、转化、还是销售支持。每种目标对应不同内容策略。

📋

内容类型匹配

基础 SEO 文章 / 判断型文章 / 权威资产页 / 转化页 / 更新复盘 / 监测报告，6 种类型各有打法。

🔍

证据先于写作

主张、证据、推理、含义、来源、新鲜度——任何一节缺失都会被 AI 标记为低质。

🛡️

商业边界先于架构

公开 / 匿名化 / 概括化 / 不公开，4 级边界决定能不能写、写到哪一层。

👤

人工审批先于发布

任何账号/CMS/API 操作前必须有人工审批节点。AI 不替你点发布按钮。

🔄

持续监测+复盘

Search Console / Analytics / 排名工具 / AI 引用数据，月度复盘形成迭代闭环。

1.2 AI SEO 内容的 6 大核心要素

从「ChatGPT 分享-AI 搜索排名算法分析」提炼的 6 大要素（资料源：30 段视频脚本深度解析）：

S

① 原创数据、真实案例、行业观察、价格/政策/流程差异

AI 最怕回答"大家都已经写过的废话"。如果你的网站有别人没有的数据（如{地区}{行业} 客户案例、{教育产品} 学生路径、费用区间、真实问答、政策变化整理、表格对比），AI 更容易引用你。这是 S 级（最值得做）。

S

② 高质量 Digital PR

不是随便发新闻稿，而是让行业媒体、协会、权威博客、学校、律师、顾问、研究报告、地方媒体提到你。AI 搜索更容易相信"第三方可信来源对你的描述"。

S

③ 品牌实体清晰度

AI 要知道：你是谁、公司叫什么、服务范围是什么、谁负责、资质是什么、和其他品牌是什么关系。尤其做多品牌、多实体的公司，如果网上信息混乱，AI 很可能不敢引用，或者引用错。

A

④ 可被 AI 抽取的内容结构

AI 喜欢清楚的定义、步骤、表格、FAQ、对比、费用、资格条件、风险提示、更新时间。不是为了"讨好 AI"，而是让答案更容易被提取。Google 官方也明确：不需要专门为生成式 AI 改写，但基础 SEO、清晰结构、独特有价值内容仍然是基础。

A

⑤ 主题权威（Topical Authority），而不是单篇爆文

不是只写一篇"什么是 {教育产品}"，而是形成完整内容群：9/10/11/12 {年级单位}路径、{评估A}、{评估B}、毕业要求、{阶段2}、费用、{形式1}、{形式2}合规、家长常见误区。AI 会更容易判断你在这个主题上"持续覆盖"。

A

⑥ 被 ChatGPT / Google / Bing 能正常抓取

非常现实。{AI 公司} 明确说，ChatGPT Search 要显示网站，重要前提是不要屏蔽 OAI-SearchBot；而 GPTBot 和 OAI-SearchBot 是不同用途，前者偏训练，后者偏搜索展示。

B

⑦ Schema / 结构化数据

适合做 Organization、LocalBusiness、FAQ、Product、Review、Article、Person、Breadcrumb。不是 AI 搜索捷径，但能帮搜索引擎理解页面。Google 官方说结构化数据可帮助理解页面和获得富结果，但不是生成式 AI 的特殊要求。

B

⑧ 真实论坛、Reddit、小红书、YouTube 评论区讨论

对品牌信任有用，但前提是自然、真实、有语境。刷 mention 没意义，甚至可能有反效果。

B

⑨ 页面速度、移动端、可访问性

不是排名核心，但如果 agent 要访问网站、读 DOM、看按钮、理解表单，网站可访问性会变重要。{AI 公司} 开发者 FAQ 也提到，ARIA 标签和可访问性有助于 ChatGPT Agent 理解网站结构和交互元素。

C

⑩ 程序化 SEO

只有在"每页有真实差异"的情况下值得做。比如"{地区} 各省 {行业} 项目对比""不同学校 {教育产品} 课程差异""{城市} 各区 {细分行业} 费用数据"。如果只是批量套模板，会越来越危险。

C

⑪ Guest post

只做高相关、高可信、有编辑审核的网站。不要做泛泛的"""Top 10" {行业} agency""垃圾站。

D

⑫ DA / DR / 低质外链 / Blog comments

可以参考，但不能当核心策略。AI 搜索更看重"这个来源是否值得被引用"，不是某个 SEO 工具给你几分。

1.3 文中没提到但同样重要的 5 个隐藏要素

💡 隐藏要素 #1：AI 引用的不是网页，是"答案片段"

你要把内容写成 AI 能直接拿来回答的问题，比如"{年份}{地区} 热门项目抽分为什么还高？""{教育产品} 11 年级入学要多久毕业？""{行业} 公司和持牌 {资质A} 的关系怎么判断？"这类内容比泛泛写品牌故事更容易被引用。

💡 隐藏要素 #2：实体一致性（Entity Consistency）

公司名、创始人、地址、电话、资质、服务范围、Logo、LinkedIn、Google Business、Crunchbase、新闻稿、社媒简介都要一致。尤其做过多品牌、多公司架构的，这种信息如果混乱，AI 会降低信任。

💡 隐藏要素 #3：监测现在开始变重要

Google 已经在 {月份}推出 Search Console 的生成式 AI 表现报告，用来查看 URL 在 AI Overviews、AI Mode、Discover 生成式功能中的曝光、页面、国家、设备和时间维度。以后 GEO 不是只靠猜，要开始看数据。

⚠️ 隐藏要素 #4：不要迷信 llms.txt

Google 明确说，Google Search 包括生成式 AI 功能不使用 llms.txt，做了不会伤害排名，但也不会帮助 Google 排名。它可以作为给其他 agent 的说明文件，但不能当成 AI SEO 的核心。

💡 隐藏要素 #5：品牌被提到的语境，比提到次数更重要

AI 不只是数 mention，还会看"谁说的、在哪里说、怎么说、是否和你的核心服务一致"。比如"X 公司是 {教育产品} 线上项目运营方"这种清楚描述，比 多个无意义品牌名出现更有价值。

AI 搜索排名的核心不是"哪里能发链接"，而是：你有没有独特、可信、结构清楚、能被第三方验证、并且能被 AI 直接拿来回答用户问题的信息。

1.4 内容生产三重审计

每篇文章必须先过审再交付，三道关卡不可省略：

✓

Trust Audit（信任审计）

可信度、来源、作者/机构、更新时间。查作者 bio、引用源、是否有同行引用。

🤖

AI Citation Audit（引用审计）

定义、表格、FAQ、步骤、结论句、可抽取性。查结构化数据、段落自包含度。

⚠️

Commercial Risk Audit（商业风险审计）

是否泄露商业机密、是否让同行可复制。查客户名单、内部 SOP、成本结构。

2 8-Agent 系统架构 8-Agent Architecture

将 SEO 专员的全部工作拆解为 8 个独立 Agent，每个 Agent 都有自己的输入/输出 schema 和检查清单。 8 个 Agent 通过 handoff（交接）机制串成完整 pipeline，失败时精准路由到对应环节而非全流程重跑。

2.2 每个 Agent 的详细职责

01

Strategy Agent · 战略

业务目标 / 受众 / 转化动作 / 禁止披露

核心职责

  • 明确商业目标：流量 / 信任 / AI 引用 / 转化 / 销售支持 / 监测
  • 锁定目标用户画像：行业、规模、痛点、决策路径
  • 设计转化动作：加微信 / 表单 / 电话 / 预约 / 下载
  • 划定商业边界：客户名单 / 内部 SOP / 成本结构 / 合作渠道
  • 选择内容类型：基础 SEO / 判断 / 权威资产 / 转化 / 更新 / 报告

输入

行业 / 网站 URL / 目标服务 / 目标客户 / 地区 / 语言 / 禁止公开信息

输出

项目 brief + 内容边界 + 合规边界 + 关键词策略

02

Research Agent · 研究

真实用户问题 / 主次问题 / SERP 竞品 / 知识模型 / 关键词冲突

核心职责

  • 调研真实用户问题（4 大意图：定义 / How-To / 列表 / 对比）
  • 设计主问题 + 3-5 个次问题（7 维评分算法选词）
  • 分析 SERP top-3 页面（能否被我的内容替代？）
  • 检查关键词冲突（{竞争现象}）
  • 建立知识模型：实体 / 关系 / 概念层级
  • 判断 DA 5 的站能打哪些词（SD ≤ 30 是 DA 较低的站 的实战阈值）
03

Evidence Agent · 证据

事实 / 数据 / 来源 / 案例 / 主张-证据-推理-含义 库

核心职责

  • 收集一手数据：行业报告 / 官方公告 / 政策文件
  • 收集二手数据：权威媒体 / 同行研究 / 学术论文
  • 建立主张-证据-推理-含义（Claim-Evidence-Reasoning-Implication）四要素
  • 维护证据库：可复用 vs 一次性
  • 检查来源新鲜度（updated_at）
04

Risk Agent · 风险

商业机密 / 隐私 / 版权 / 平台条款 / 法律 / 品牌

核心职责

  • 扫描商业机密泄露风险
  • 检查隐私数据（PII）
  • 验证版权和引用授权
  • 审核平台服务条款（robots.txt、OAI-SearchBot、GPTBot）
  • 法律风险（如医美 / 金融 / {行业}行业的合规要求）
  • 品牌口径一致性（多品牌实体）

输出

风险评估：🟢 通过 / 🟡 需修改 / 🔴 高风险暂停

05

Content Architect Agent · 内容架构

内容蓝图 / 章节 / 表格 / FAQ / CTA / 内链 / Schema / AI 可抽取结构

核心职责

  • 设计 9 节均衡结构（问题→答案→证据→对比→FAQ→CTA）
  • 布局 JSON-LD：Article + FAQPage + BreadcrumbList + Organization
  • 段落自包含（每段独立有意义）
  • 内链网：分类页 → 文章 → 相关文章 3 层结构
  • CTA 设计：上下文匹配（不要硬塞转化）
  • 视觉资产：表格 / 图表 / 截图（不是装饰，是信息）
06

Writer Agent · 写作

草稿 / 标题 / meta / FAQ / 摘要 / 平台变体 / 去 AI 腔

核心职责

  • 按公式写：具体人群 + 具体场景 + 具体问题 + 企业经验 + 数据/案例 + 风险判断 + 可执行建议
  • 避免 AI 腔（"在当今时代""毫无疑问""总而言之"）
  • 避免 clickbait（夸张、虚假承诺）
  • 生成平台变体：网站 / 公众号 / 小红书 / 知乎 / Medium
  • 标题/description/FAQ/excerpt 一并输出

核心公式

具体人群 + 具体场景 + 具体问题 + 企业经验 + 数据/案例 + 风险判断 + 可执行建议
07

Audit Agent · 审计

SEO / 语义 / 信任 / AI 可抽取 / 商业风险 / 转化审计 / 失败路由

核心职责

  • 执行 19 规则 × P0-P3 风险分级（详见第 3 章）
  • 三重审计：Trust + AI Citation + Commercial Risk
  • 失败路由：精准回到对应 Agent（不重跑全流程）
  • 输出 7 段结构化报告
08

Monitor Agent · 监测

Search Console / Analytics / 排名 / AI 引用 / 转化 / 用户反馈

核心职责

  • GSC 28 天滚动数据：曝光 / 点击 / CTR / 排名
  • AI 引擎引用监测：{AI 引擎1} / {AI 引擎2} / {AI 引擎3} / AI Overviews
  • 排名变化追踪：每周 + 每月复盘
  • 用户反馈收集：评论 / 表单 / 客服
  • 更新 backlog：哪些文章该刷新

3.3 7 段结构化报告模板

每次审计输出 7 段标准报告：

  1. Executive Summary — 1 段话讲清整体健康度
  2. P0/P1/P2/P3 风险表 — 按风险等级分组的 findings
  3. 证据溯源 — 每个 finding 的 URL/HTML 片段/截图
  4. 修复建议 — 具体到命令级别
  5. 修复后效果预期 — GSC 指标变化预测
  6. 附录：原始数据 — 可被审计的原始 JSON
  7. 下阶段计划 — 修复后进入 Phase 9

5 9 节均衡写作 + 4 层内容分层 Content Writing

不是"写 3000 字"——而是按 9 节均衡结构 写，确保每节都有信息密度。 内容按 4 层分层：基础事实 → 判断 → 经验 → 风险，构成信息增益。

5.1 9 节均衡结构

每篇文章必须覆盖这 9 个章节（可以合并但不能缺失）：

1

Hook（钩子）

3 秒抓住读者：用户痛点 / 反常识 / 紧迫性

2

Definition（定义）

用 1 段话讲清"这是什么"，AI 最爱引用

3

Context（背景）

为什么这个问题重要？时间线 / 市场规模 / 影响范围

4

Method / How-to（方法）

步骤化、可执行；用编号列表，不是段落

5

Comparison（对比）

表格化比较至少 3-5 个方案

6

Evidence（证据）

数据 / 案例 / 引用源 / 截图 / 时间戳

7

FAQ（常见问题）

4-6 条 Q&A，每条 1-3 段，AI 抽取频率最高

8

Risk（风险）

副作用 / 反例 / 常见错误 / 何时不应该这么做

9

CTA + Related（行动 + 相关）

明确的下一步 + 3-5 个相关文章内链

  2. 和本文相似的 top-3 文章，他们没写什么？

5.3 AI 可引用结构审计 8 项

每篇文章必须通过这 8 项结构审计：

  1. 每段独立可读 —— 段落自包含，不依赖上下文
  2. 定义清晰 —— 第一段就讲清"这是什么"

7 外链 / 负面 SEO / Spam Backlinks Backlinks & Negative SEO

垃圾外链不会直接让网站降权（Google Penguin 后），但仍需监控。 负面 SEO 攻击有 4 大类型，识别后必须 4 步处理。 来源：「ChatGPT 分享-垃圾外链与负面SEO」48KB 深度分析。

7.1 4 大垃圾外链类型

🚜

① Link Farm（链接农场）

大量低质网站互相链接形成网状结构，无实际内容。识别：IP 段集中 / 域名年龄新 / 出站链接数 > 100

🤖

② Auto-generated Sites（自动生成站）

用 GPT/AI 批量生成的内容站，无人工编辑。识别：内容无内在逻辑 / 互相抄袭 / 模式化标题

⚠️

③ Negative SEO（负面 SEO 攻击）

竞争对手故意给你发垃圾外链。识别：突然大量异常外链 / 锚文本异常 / 集中在某关键词

💬

④ Blog Comments / Forum Spam

博客评论、论坛签名大量留链接。识别：锚文本高度重复 / 来自不相关主题站点

7.2 关键区分：4 个相关概念

概念定义Google Penguin 后是否还有效？
Link Farm一群专门为互相链接存在的网站无效（Google 早就能识别）
PBN (Private Blog Network)自己控制的私人博客网络，互相链接无效 + 高风险（一旦被识别会牵连整站）
Spam Backlinks来自垃圾站的外链无害（被 Google 忽略）
Toxic Backlinks人工处罚级别的外链（少见）有害 → 需 disavow
Negative SEO Attack竞争对手故意制造垃圾外链Google 否认有效，但实测可能短期波动

7.3 Google Penguin 算法的核心规则

Google Penguin 2012 后：大部分垃圾外链被算法自动忽略，不会传递权重，也不会降权。 只有在极端情况下（人工处罚 / 模式明显恶意）才需要 disavow。

不会降权的情况（不用管）

  • 少量博客评论外链（10-100 个）
  • 来自低 DA 站点的链接
  • 锚文本不完全匹配的链接
  • 来自无关行业的链接
  • PBN 链接（如果 Google 没识别，就当"普通"链接）

可能降权的情况（建议处理）

  • 几天内突然增加 {R}+ 异常外链（可能是负面 SEO 攻击）
  • 锚文本 高比例匹配你的核心商业关键词（操控迹象）
  • 来自明显垃圾站的成千上万个链接
  • GSC 收到 "Unnatural links" 手动操作警告
  • 曾经买过链的历史（历史负担）

7.4 处理方案 3 选 1

方案 1（推荐）：观察为主

  1. 用工具（第三方外链分析工具）监控新外链
  2. 设置阈值告警：单日新增 > 50 触发
  3. 每周看 GSC "Links" 报告
  4. 没有手动操作警告就不动

方案 2：Disavow 部分链接

  1. 在工具中识别最严重的 Spam 来源（域名级）
  2. 列出 .txt 格式的 disavow 列表：domain:spamlink1.com
  3. 在 GSC 上传 disavow 文件
  4. 等 1-2 个月观察

方案 3：完全 disavow（仅紧急情况）

  1. 在 GSC 收到 manual action 时使用
  2. 整站或大面积 disavow
  3. 提交 reconsideration request

7.5 关键原则

⚠️ 不轻易 disavow

大部分垃圾外链先监控，不轻易 disavow；只有出现人工处罚、明显恶意模式或历史买链风险时再建议进一步处理。

💡 优先级

把 80% 时间放在内容质量 + Digital PR + 实体清晰度，把 20% 时间放在外链监控。不要本末倒置。
