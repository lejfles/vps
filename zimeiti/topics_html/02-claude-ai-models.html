Claude / AI 模型与产品

百科 + 工具 + 数据 · 从 Haiku 到 Opus 的模型矩阵指南
最后更新：2026-07-07 · 数据来源：公开 web 报告 + 真实工具平台 + 视频脚本观点补充

这是什么：LLM（Large Language Model，大语言模型）产品矩阵，包括 Claude（Anthropic）、ChatGPT（OpenAI）、Gemini（Google）等。本话题涵盖模型分层、Agent 形态、定价、安全风险、AI 写作质量、第二大脑工作流。

为什么重要：2026 年中主力推理模型单 token 成本相比 2024 年下降 70%。Claude Sonnet 5 SWE-bench Verified 达 72.7%，Opus 4.8 达 79.4%。AI 产品的形态从单一聊天窗口扩展到多产品矩阵（Haiku 实时 / Sonnet 主力 / Opus 深度），企业级部署依赖任务路由。

给谁看：AI 应用开发者、企业决策者、产品经理、内容工作者、AI 安全工程师。

一、核心概念

1.1 模型三层架构

层级代表模型用途成本
实时推理Haiku 4.5 / GPT-4o-mini高频、低延迟、成本敏感主力 1/5–1/20
主力推理Sonnet 5 / GPT-4o大多数企业级任务$3/$15 per M tokens
深度推理Opus 4.8 / o1复杂规划、深度分析$5/$25 per M tokens

1.2 AI Agent 七种形态

基于 Anthropic《Building Effective Agents》和创作者描述：

  1. 检索型：单次/多次搜索 + 总结（竞品监测、研究助手）
  2. 工具调用型：调用外部 API 执行操作（CRM 更新、订单处理）
  3. 规划型：长链条规划 + 多步执行（项目管理、研究计划）
  4. 协作型（Multi-Agent）：多个 Agent 角色分工（内容生产流水线、决策支持）
  5. 代码型：自主编写、调试、执行代码（自动化运维、数据处理）
  6. 监控型：长期运行，持续观察并响应事件（品牌舆情、SRE 监控）
  7. 自主学习型：从执行结果中持续优化策略（推荐系统、广告投放）

1.3 任务路由

把 80% 轻量任务用 Haiku，20% 复杂任务用 Opus，总体成本可比"全主力"降低 60% 以上。Stanford HELM 评测显示复杂规划任务 Opus 比 Haiku 准确率高 47%，但成本高 18 倍。

2.4 Prompt Injection 安全风险

  • OWASP 2025 Top 10 for LLM Applications：Prompt Injection 排 #1
  • OWASP LLM Top 10 包括：Prompt Injection, Insecure Output Handling, Training Data Poisoning, Model DoS, Supply Chain, Sensitive Information Disclosure 等
  • MITRE ATLAS 互补：用于威胁建模和红队规划
  • 核心问题：当前 LLM 处理输入的方式使得 prompt injection 是内在属性，不是可修补的 bug

四、方法论：从入门到进阶

4.1 入门（个人使用）

  1. 用 ChatGPT 或 Claude 网页版替代 Google 搜索，处理日常问题
  2. 学习"答案前置"提问法：直接给上下文 + 具体问题
  3. 建立"第二大脑"：把读过的资料、笔记向量化存储

4.2 中级（API 集成）

  1. 选择任务路由架构（Haiku 处理高频简单任务，Sonnet/Opus 处理复杂任务）
  2. 用 LangChain 或直接 API 构建第一个 Agent
  3. 建立 prompt 模板库和评测数据集

4.3 进阶（生产级）

  1. 实施 Prompt Injection 防御（输入过滤、人类审批节点、工具白名单）
  2. 用 MCP 连接企业内部数据源
  3. 建立成本监控和用量限额（防止 Agent 循环失控）

五、典型应用场景

5.1 客服自动化

用 Haiku 4.5 处理 80% 常见问题，Opus 处理 20% 复杂问题。某创作者案例：4 分钟内主动触达热点线索，转化率提升 30%–80%。

5.2 代码助手

Sonnet 5 SWE-bench 72.7%，接近 Opus 4.8（79.4%）但成本 40%。领先做法：日常编码用 Sonnet 5，关键 bug fix 用 Opus。

5.3 内容生产流水线

多 Agent 协作：检索 Agent（找资料）+ 写作 Agent（基于检索写稿）+ 编辑 Agent（润色）。每个 Agent 用合适的模型层。

六、常见误区

6.1 "一刀切用一个模型"

反模式。任务路由能让成本降低 60%。

6.2 "Agent = prompt 工程"

Agent 的核心是有目标、能感知、能行动、能反馈。没有反馈循环的"Agent"实质上还是 prompt。

6.3 "AI 写作完美无瑕"

反之，AI 内容识别难度增加但被识别为低质量概率也在上升。核心方法：加入具体场景判断、第一人称经历、可被反驳的具体观点。

6.4 "Prompt Injection 是边缘风险"

错。OWASP 2025 把它列为 #1。一份中毒文档可影响所有 AI 用户。

七、进阶话题

7.1 Multi-Agent 协作的经济学

多个 Agent 角色协作是否真有生产力提升，还是把单个 Agent 的复杂度分散到多个？创作者观点不一，需要根据任务特征决定。

7.2 按结果计费模式

部分代码 Agent 按成功执行的 bug 修复数收费。这是否会取代按 token 计费？2026 年仍处于早期阶段。

7.3 垂直行业模型突破

医疗、法律、金融行业的专用模型能否突破通用模型的能力上限？Fable 系列是早期尝试。
